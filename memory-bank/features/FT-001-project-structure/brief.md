# FT-001: Структура проекта

Issue: https://github.com/4itosik/feedium/issues/1

## Problem

В репозитории 0 строк исполняемого кода — только документация. Любая следующая фича не может быть начата: некуда добавить хендлер, провайдер или worker. Ближайшая заблокированная фича — Health Check (FT-002), которая должна подтвердить, что процесс жив и конфигурация загружена.

## Task

Разблокировать работу над прикладными фичами: у разработчика и AI-агента должно быть очевидное место, куда класть новый код, без необходимости каждый раз заново принимать архитектурные решения, уже зафиксированные в memory-bank.

## Context

- Следующая запланированная фича — Health Check (FT-002); её нельзя начать, пока нет точки входа, конфигурации и серверного слоя
- Архитектура, стек и конвенции уже зафиксированы в [architecture.md](../../domain/architecture.md) и [coding-style.md](../../engineering/coding-style.md) — задача исполнительская, не проектная
- Откладывать нельзя: без каркаса каждая следующая фича будет тянуть за собой споры о структуре, которых memory-bank как раз и должен был избежать

## Users

Backend-разработчики проекта и AI-агенты, реализующие последующие фичи.

## Scope

### In Scope

- Каркас проекта, в котором можно начать реализацию FT-002 (Health Check) без предварительных архитектурных решений
- Точка входа, загрузка конфигурации, запуск и корректная остановка процесса
- Совместимость с существующими `.golangci.yml` и CI

### Out Of Scope

- Сам эндпоинт Health Check и любая прикладная логика
- Бизнес-логика (коллекторы, скоринг, ранжирование)
- Интеграции с внешними сервисами (Telegram, RSS, AI)
- Доменные модели и схема БД

## Acceptance Criteria

- `go build ./...` завершается без ошибок
- `golangci-lint run ./... -c .golangci.yml` завершается без ошибок
- Процесс запускается, доходит до состояния «готов принимать запросы» и корректно останавливается по SIGTERM (код выхода 0, без зависших горутин)
- FT-002 (Health Check) может быть реализован добавлением одного нового компонента и его регистрацией в DI — без изменений в точке входа и в загрузке конфигурации
- Структура каталогов соответствует [coding-style.md](../../engineering/coding-style.md)

## Pre-existing Assets

- Документация архитектуры и coding style (memory-bank) — создана, не дублировать
- `.golangci.yml` — уже в репозитории
- CI (`ci.yml`) — уже есть
