# Project Problem Statement

Feedium — персональный агрегатор контента для одного пользователя. Система собирает посты из Telegram, RSS и веб-источников, обрабатывает их через AI и отдает одну отранжированную ленту вместо ручного обхода десятков каналов.

Документ фиксирует project-wide контекст, который не должен дублироваться в feature- и implementation-документах.

## Boundary With PRD

- `domain/problem.md` — устойчивый контекст Feedium: зачем существует продукт, какие workflows критичны, какие ограничения не должны ломаться.

## Product Context

Основной пользователь один: владелец системы. Целевая задача — убрать ручной обход 10+ источников и получать релевантную дневную ленту из одной точки.

Проблема текущего процесса: высокий шум (80%+ нерелевантного), много переключений контекста между Telegram, сайтами и RSS, отсутствие единого ранжирования по интересам.

Границы продукта:
- solo-use, без мультитенантности;
- фокус на агрегации и ранжировании, а не на публикации контента;
- API-first backend и web UI; OpenClaw-бот выступает вторым интерфейсом потребления/управления.

## Core Workflows

- `WF-01` Ingestion: встроенные коллекторы (Telegram userbot, RSS, web scraping) забирают контент и нормализуют его в универсальную модель поста.
- `WF-02` Enrichment and ranking: посты сохраняются, затем проходят AI-обработку (summary/score) и ранжирование по интересам.
- `WF-03` Consumption: пользователь открывает web UI или OpenClaw и получает приоритизированную ленту без ручного обхода исходных источников.

## Constraints

- `PCON-01` Single-user: система проектируется под одного пользователя без tenant-модели и ролей.
- `PCON-02` Telegram userbot несет операционный риск (rate limits/бан), поэтому критичен контроль частоты и изоляция аккаунта.
- `PCON-03` Стоимость и латентность LLM ограничивают глубину и частоту AI-процессинга.
- `PCON-04` Веб-скрапинг хрупкий к изменению верстки; при наличии RSS предпочтителен RSS.
- `PCON-05` В MVP обрабатывается в первую очередь текстовый контент.
- `PCON-06` React UI в MVP минимальный и подчинен главной цели: быстрое потребление отранжированной ленты.
